# 出勤打卡系統使用手冊（實習生版）

> 版本：v2.0
> 更新日期：2025-11-18

---

## 1. 系統概觀

本系統提供以下功能：

- 上下班與中途外出打卡
- 請假與補打卡申請
- 實地 / 遠端實習打卡與統計
- 個人出勤紀錄與扣分查詢
- 自動更新 Google 行事曆與 Notion 請假看板（經主管核准後）
- Discord Bot 整合（打卡、查詢、提醒）

---

## 2. 登入與個人專屬頁

### 2.1 Token 驗證方式（第一階段）

每位實習生會分配到一個專屬連結，例如：

```
https://timeclock.vdo.tw/I86?token=abc123xyz
```

- `I86` 為你的實習代碼
- `token` 為個人專屬 token，定期更換以確保安全
- 點擊該網址即可進入個人出勤頁面

> **安全注意事項**：
> - 請勿將連結轉給非本團隊人員
> - 若發現 token 外洩，請立即通知管理員重置

### 2.2 進階登入（未來規劃）

未來將啟用 Discord OAuth2 或 Google OAuth2 登入功能。

## 3. 日常打卡操作

### 3.1 上班打卡

**實地上班時**：
- 方式一：到公司後，開啟個人頁面按「實地上班打卡」
- 方式二：在 Discord 使用 `/in onsite` 指令

**遠端上班時**：
- 方式一：到個人頁面按「遠端上班打卡」
- 方式二：在 Discord 使用 `/in remote` 指令

**成功後顯示**：
- 打卡時間
- 出勤類型（實地 / 遠端）
- 當日班表資訊

> **提醒**：實地上班提供交通津貼，遠端上班則無。實習證明會分開列出兩種工時。

### 3.2 下班打卡

結束當日工作時：
- 到個人頁面按「下班打卡」，或使用 `/out` 指令

**系統會顯示**：
- 今日工時（已自動扣除 1.5 小時午休）
- 今日是否遲到
- 是否早退
- 是否有異常（缺打卡、工時不足等）
- 預估扣分與原因（若有）

### 3.3 中途外出（扣除時數）

若需離開工作崗位（不論實地或遠端）：

**離開前**：
- 按「休息開始」或使用 `/break_start`

**回來後**：
- 按「休息結束」或使用 `/break_end`

系統會依「休息開始」到「休息結束」的時間，扣除實際工作時數。

### 3.4 午休

- **午休時間**：12:00 - 13:30（固定 1.5 小時）
- **不用打卡**：系統會自動扣除

## 4. 出勤型態說明

### 4.1 實地上班（Onsite）

- 在辦公室工作
- 提供交通津貼
- 實習證明會列出實地工時

### 4.2 遠端上班（Remote / WFH）

- 在家或其他地點遠端工作
- 未來主要以遠端實習為主
- 無交通津貼
- 實習證明會分開列出遠端工時

> **重要**：打卡時請正確選擇出勤型態，系統會分開統計以便計算津貼與製作實習證明。

## 5. 請假流程

### 5.1 預先告知（強烈建議）

**為何要預先告知？**
- 免附請假證明
- 主管可提早安排工作
- 減少扣分風險

**如何預先告知？**
1. 確認你可能無法出席的日期與時間
2. 儘早透過以下管道告知主管：
   - Discord 私訊或指定頻道
   - Teams 訊息
   - 打卡系統的「預先告知」功能
3. 內容包含：
   - 預計請假日期與時段
   - 簡要原因

**建議時限**：
- 1 日內請假：至少前 1 個工作日告知
- 2 日以上請假：至少前 3 個工作日告知
- 1 週以上請假：至少前 1 週告知

### 5.2 正式請假申請

預先告知後，仍需透過系統提交正式請假申請：

1. 到個人頁面，點選「請假申請」
2. 填寫：
   - 起始日期與時間
   - 結束日期與時間
   - 請假類型（病假 / 生理假 / 事假 / 其他）
   - 請假原因描述
3. 若有證明文件，可直接上傳：
   - 系統會自動存到公司 Google Drive

**審核流程**：
- 主管審核通過後：
  - 自動寫入系統
  - 自動更新 Google Calendar
  - 自動同步 Notion 請假看板
- Discord Bot 會通知審核結果

### 5.3 當日臨時請假

**適用情況**：突發事件（生病、家庭事故等）

**緊急處理步驟**：
1. 儘速透過任何管道告知主管（電話、訊息優先）
2. 儘快登入系統填寫請假單
3. 上傳證明文件（醫院證明等）

> **注意**：當日臨時請假可能依規定扣分，除非有明確突發事件證明。

## 6. 補打卡流程

### 6.1 什麼時候需要補打卡？

- 今天有到場 / 有工作，但忘記上班或下班打卡
- 中途外出沒有按「休息開始 / 結束」
- 系統明顯異常（操作了但沒有記錄）

### 6.2 如何補打卡？

1. 到個人頁面 → 點選「補打卡申請」
2. 填寫：
   - 補打卡日期、時間
   - 類型（上班 / 下班 / 休息 / 實地 / 遠端）
   - 原因說明
   - **「下次如何避免」改善方案**（必填欄位）
3. 若有相關證明（截圖、記錄等），可一併上傳

**審核流程**：
- 主管審核通過後，系統會補寫該筆出勤記錄
- 會重新計算當日工時與扣分
- Discord Bot 會通知審核結果

> **重要**：補打卡偶爾可接受，但過多（每月 > 3 次）會被視為管理問題並扣分。

## 7. 班表管理

### 7.1 查看個人班表

在個人頁面可以看到：
- 每週固定班表（例如：週一至週五 08:30-18:00）
- 特殊調整（例如：週三因課程改為 10:00-18:00）
- 實習期間（起始日至結束日）

### 7.2 班表異動申請

**長期異動（推薦）**：
- 情況：下學期課表改變，需要固定調整某天的時間
- 做法：至少 **1 個月前**告知主管或人資
- 流程：管理員調整班表 → 產生預覽 → 你確認 → 生效
- 優點：調整後的時段不算班表時間，**不需請假**

**短期臨時變動**：
- 情況：某天臨時有事，需要變更時間
- 做法：走請假流程
- 影響：會計入請假統計，影響出勤率

## 8. 查看個人出勤與扣分

### 8.1 個人頁面功能

打開你的個人頁面（例如 `https://timeclock.vdo.tw/I86?token=xxx`），可以看到：

**今日視圖**：
- 今日打卡記錄（上班、下班、休息）
- 實時工時計算
- 是否遲到 / 早退
- 當日扣分（若有）

**歷史視圖**：
- 選擇日期範圍查看過往出勤
- 遲到 / 早退 / 請假記錄
- 每日工時統計（實地 / 遠端分開顯示）

**當月統計**：
- 當月出勤天數
- 實地工時 vs 遠端工時
- 遲到次數
- 請假次數
- 補打卡次數
- **當月出勤分數**與扣分明細

**扣分明細**：
- 扣分原因（遲到、早退、補打卡等）
- 扣分時間與點數
- 是否有預先告知
- 改善狀態

## 10. 特殊情況處理

### 10.1 颱風假

**實地上班者**：
- 政府宣布停班停課 → 可不上班，不需請假
- 可選擇改為遠端工作（需事前告知）

**遠端工作者**：
- 原則上仍需正常上班
- 若受颱風影響（停電、網路中斷）→ 可請假並說明原因

### 10.2 系統異常

若遇到以下情況：
- 打卡按鈕無反應
- 時間顯示錯誤
- 無法載入頁面

**處理步驟**：
1. 截圖保存證據
2. 立即透過 Discord/Teams 告知主管與技術人員
3. 系統恢復後，透過補打卡流程補登
4. 上傳截圖證明

> 系統異常導致的補打卡，只要有證明，不會扣分。

### 10.3 忘記帶裝置或無法使用

若在公司但無法使用個人裝置打卡：
- 可借用同事裝置登入（使用你的 token 連結）
- 或當日結束後立即補打卡並說明原因

## 11. 常見問題（FAQ）

**Q1：忘記打卡但有確實上班，會被當作缺勤嗎？**
- A：不會，但你必須盡快填補打卡申請，說明原因與改善方式。若常常忘記打卡，會依規定扣分。

**Q2：已經預先告知會遲到，還會被扣分嗎？**
- A：每月前 3 次且 ≤ 30 分鐘不會扣分。但若連續幾個月都頻繁遲到（即使預先告知），系統會啟動防濫用機制。

**Q3：請假一定要附證明嗎？**
- A：若有事前預告且天數不長，通常不需證明。但當日臨時請假或請假次數過多，主管有權要求證明。

**Q4：我認為扣分不合理，可以申訴嗎？**
- A：可以。請先整理實際情況與相關證據，與主管面談討論。系統會依討論結果調整記錄。

**Q5：實地和遠端打卡有什麼差別？**
- A：實地上班提供交通津貼，遠端則無。實習證明會分開列出兩種工時統計。請務必正確選擇打卡類型。

**Q6：早退會很嚴重嗎？**
- A：是的。早退被視為態度問題，第一次扣 3 分，第二次起扣 5 分。若確實需提前離開，務必事前告知並請假。

**Q7：什麼是「預先告知」？一定要用嗎？**
- A：預先告知是在上班前 30 分鐘告知可能遲到或請假。雖非強制，但可享有免證明、減扣分等優惠，強烈建議使用。

**Q8：Token 連結遺失了怎麼辦？**
- A：請聯繫管理員重新發送。為了安全，不建議將 token 儲存在公開位置。

## 附錄：快速操作指南

### 每日標準流程

```
08:30 前抵達 → 開啟個人頁面或 Discord
→ 點選「實地上班打卡」或 `/in onsite`
→ 系統顯示打卡成功

工作中...

若需外出 → 按「休息開始」或 `/break_start`
回來後 → 按「休息結束」或 `/break_end`

18:00 工作結束 → 按「下班打卡」或 `/out`
→ 系統顯示今日工時與狀態
```

### 預先告知流程

```
前一天或當天上班前 30 分鐘
→ Discord 私訊主管或使用 `/advance_notice`
→ 說明：「明天可能會遲到 15 分鐘，因交通因素」
→ 當天實際情況發生時正常打卡
→ 系統自動比對預先告知記錄，給予優惠
```

### 請假流程

```
確定需要請假（至少提前 1 天）
→ Discord/Teams 先告知主管
→ 登入個人頁面 → 「請假申請」
→ 填寫完整資訊 → 上傳證明（若需要）
→ 提交申請
→ 等待主管審核
→ Discord Bot 通知審核結果
→ 若通過，自動同步 Google Calendar 與 Notion
```
